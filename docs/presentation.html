The Problem

We use AI assistants for clinical data engineering, reporting, and security guidance.
But how do we know the answers are correct, compliant, and safe?

Compliance Risk

  • Does the AI cite correct FDA regulations?
  • Does it recommend audit trails per 21 CFR Part 11?
  • Does it enforce HIPAA Safe Harbor de-identification?

No Standard Measurement

  • Different team members get different quality answers
  • No way to compare models objectively
  • No tracking of quality over time

Prompt Engineering is Guesswork

  • System prompts are written intuitively
  • No data on what works vs what doesn't
  • No A/B testing of prompt variations

What We Evaluate (Section 1)

  • Prompt Quality — Does the model cite SDTM IG v3.4? Recommend Pinnacle 21? Enforce audit trails?
  • Category Scores — Regulatory, CDISC, Pipeline, Security, GDPR, AI Governance
  • Recommendations — Specific prompt improvements to fix weak areas

How It Performs (Section 2)

  • Latency — Avg, P95, min/max response time
  • Token Usage — Input/output tokens, verbosity
  • Cost — Estimated API cost per evaluation run

Clinical Trials Roles & Criteria

Test criteria derived from the DCRI Clinical Trials Test Suites document — mapped to specific regulations.

Clinical Data Engineer

12 tests | 60 criteria

  • Audit trails (21 CFR §11.10(e))
  • SDTM IG v3.4 + Pinnacle 21
  • ADaM IG v1.3 derivations
  • HIPAA Safe Harbor (18 identifiers)
  • SAS-to-Spark parity validation
  • GDPR cross-border transfers

Clinical Reporting Engineer

10 tests | 50 criteria

  • DSMB safety reports + data locks
  • Double-programming verification
  • TLF tolerance thresholds
  • eCTD submission (XPT v5)
  • SAE signal detection
  • Report versioning (§11.50)

Clinical Security Engineer

10 tests | 50 criteria

  • PHI pseudonymization + Key Vault
  • Unblinding controls (ICH §5.5.2)
  • AES-256 / CMK / FIPS 140-2
  • VNet isolation (NIST 800-53)
  • EU-US transfer mechanisms
  • Immutable audit logging

+ 4 data platform roles: Azure Data Engineer, Fabric Data Engineer, SQL Report Engineer, Power BI Engineer

Key Capabilities & Next Steps

What We Have Today

  • 7 roles — 3 clinical + 4 data platform
  • BYOK — OpenAI, Claude, Azure, Foundry, Gemini, Ollama
  • Web dashboard — run evaluations, A/B comparisons, track history
  • DeepEval GEval — LLM-as-judge with chain-of-thought scoring
  • Performance metrics — latency, tokens, cost tracking
  • DCRI criteria — mapped to FDA, HIPAA, GDPR, CDISC standards
  • Flexible comparisons — different models, prompts, or both
  • Separate judge model — independent scoring for unbiased evaluation

Potential Next Steps

  • Expand test suites — more clinical scenarios from DCRI SOPs
  • Automated regression — run on schedule, alert on score drops
  • Model comparison matrix — rank multiple models side-by-side
  • Prompt auto-improvement — use recommendations to auto-refine prompts
  • Integration with CI/CD — block deployment if prompt scores drop below threshold
  • RAG evaluation — test retrieval-augmented generation pipelines
  • Custom criteria builder — UI for non-technical users to add test cases
